24 /*
25  GFONTDEVICE define here is an application specifc optimized type
26  to identify the specific external device where the virtual font
27  buffer is located.
28 
29  A GFONTDEVICE type parameter is incorporated in each virtual font
30  structure as device_id and which are passed as a parameter in the
31  getvmem(..) call.
32  The parameter type and its value is transperant (don't care) for the
33  graphic library.
34 
35  Typical use for GFONTDEVICE could be
36  - the physical address for the memory chip /CS
37  - the SPI address for a memory device on a SPI bus
38  - an index into an array of device access information
39  - a const pointer to a { filename, FILE * } structure
40  etc
41 
42  A SGUCHAR type is used as default to get a minimum font memory footprint if the
43  vf_device parameter is not used by the getvmem(..) function implementation
44 */
45 typedef SGUCHAR GFONTDEVICE;
46 /*typedef SGULONG GFONTDEVICE;*/ /* ex processor chip select address */

49  Get data from virtual memory

52  vf_device= device identifier (ex address) for virtual font.

56  index = source index for data in virtual font to be loaded to RAM
